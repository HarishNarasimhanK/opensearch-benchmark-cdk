# OpenSearch Profiler Stack

CDK stack that provisions two EC2 instances: one builds OpenSearch from source with the datafusion engine, and a benchmark instance that runs OpenSearch Benchmark (OSB) against the clickbench workload.

## Prerequisites

- [Node.js](https://nodejs.org/) 18+ and npm (Node 16 works if you remove the `throughput` parameter on line 93 of `lib/opensearch-codeguru-stack.ts`)
- [AWS CLI v2](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)
- [jq](https://jqlang.github.io/jq/download/) (used by `deploy.sh` to parse CDK outputs)
- AWS credentials configured (`aws configure` or your organization's credential tool)
- CDK bootstrap completed in your target account/region:
  ```bash
  npx cdk bootstrap aws://<ACCOUNT_ID>/<REGION>
  ```

> No global AWS CDK install is needed — `aws-cdk` is included as a devDependency and runs via `npx`.

## Quick Start

```bash
git clone ssh://git.amazon.com:2222/pkg/OpenSearchEC2WithCodeGuruCDK
cd OpenSearchEC2WithCodeGuruCDK
git fetch origin share/hxarishk/hxarishk/automating-performance-correctness-tests
git checkout hxarishk/automating-performance-correctness-tests
npm install
npx cdk bootstrap   # only needed once per account/region
npx cdk deploy
```

If no `.env` file exists, `setup-env.sh` runs automatically to discover your default VPC, subnet, security group, key pair, and S3 bucket, then writes `.env`.

CDK outputs will show SSH commands and instance details for both instances.

### Using the deploy script

`deploy.sh` wraps `cdk deploy` and prints connection details after deployment:

```bash
bash scripts/deploy.sh
```

## Architecture

| Instance | Default Type | EBS | Purpose |
|---|---|---|---|
| OpenSearch | `r7g.2xlarge` | 100 GB gp3 | Builds OpenSearch from source, installs plugins, starts OpenSearch |
| Benchmark | `m7g.medium` | 500 GB gp3 | Installs OSB, clones workload repo, auto-runs clickbench benchmark |

Both instances share the same VPC, subnet, and security group. The benchmark instance communicates with OpenSearch via private IP on port 9200.

The stack creates its own IAM role with `AmazonS3FullAccess` and `CloudWatchAgentServerPolicy` — no pre-existing instance profile is required.

## SSH Key Pair

On first run, `setup-env.sh` creates an EC2 key pair called `opensearch-benchmark` and saves the private key to `$HOME/opensearch-benchmark.pem`. Keep this file safe — it cannot be downloaded again. If lost, delete the key pair in AWS and re-run `setup-env.sh`.

## Configuration

| Variable | Description |
|---|---|
| `CDK_ACCOUNT` | AWS account ID |
| `CDK_REGION` | AWS region |
| `VPC_ID` | VPC to launch instances in |
| `SUBNET_ID` | Public subnet (must auto-assign public IPs) |
| `SUBNET_AZ` | Subnet availability zone |
| `SECURITY_GROUP_ID` | Security group allowing SSH (22) and OpenSearch (9200) |
| `KEY_PAIR_NAME` | EC2 key pair name |
| `PEM_PATH` | Local path to the `.pem` private key |
| `S3_PROFILE_BUCKET` | S3 bucket for flamegraphs and benchmark results |

## Logs

### OpenSearch Instance

```bash
ssh -i $HOME/opensearch-benchmark.pem ec2-user@<opensearch-public-dns>
```

| Log | Description |
|---|---|
| `/var/log/user-data.log` | Build progress — clone, gradle, plugin install, OpenSearch start |
| `~/opensearch-run.log` | OpenSearch runtime — errors, queries, GC |
| `~/profile-cron.log` | Async-profiler cron output |

### Benchmark Instance

```bash
ssh -i $HOME/opensearch-benchmark.pem ec2-user@<benchmark-public-dns>
```

| Log | Description |
|---|---|
| `/var/log/user-data.log` | Setup progress — pip install, git clone, OSB install |
| `~/benchmark-run.log` | Benchmark execution — wait for OpenSearch, indexing, query results |
| `~/.osb/logs/benchmark.log` | Detailed OSB logs — per-query errors, HTTP responses |
| `~/benchmark-results/*.csv` | Final benchmark results |

### Troubleshooting

```bash
# OSB query errors (benchmark instance)
grep "ERROR" ~/.osb/logs/benchmark.log | grep -v "error occured" | tail -20

# OpenSearch engine errors (OpenSearch instance)
grep -A 10 "ERROR" ~/opensearch-run.log | tail -40

# Test a PPL query manually
curl -s -X POST 'http://<opensearch-private-ip>:9200/_plugins/_ppl' \
  -H 'Content-Type: application/json' \
  -d '{"query": "source = clickbench | stats count()"}' | python3 -m json.tool
```

## Benchmark Results

Results are saved to:
- Instance: `~/benchmark-results/<run-id>.csv`
- S3: `s3://<s3-profile-bucket>/benchmark-results/<run-id>.csv`

Download to your machine:
```bash
aws s3 cp s3://<s3-profile-bucket>/benchmark-results/ ./results/ --recursive
```

## Useful Commands

```bash
# CDK
npx cdk deploy                  # Deploy stack (auto-runs setup-env.sh if no .env)
npx cdk destroy                 # Tear down all resources
npx cdk synth                   # Preview CloudFormation template
bash scripts/deploy.sh          # Deploy + print SSH/connection details

# On the OpenSearch instance
curl -s http://localhost:9200                  # Health check
./opensearch/bin/opensearch                    # Start OpenSearch manually
./profile-opensearch.sh                        # One-off CPU flamegraph
ls ~/profiles/                                 # Local flamegraphs

# On the Benchmark instance
bash run-benchmark.sh                          # Run benchmark (auto-runs on deploy)
bash run-benchmark.sh <opensearch-private-ip>  # Target a specific host
cat ~/benchmark-results/*.csv                  # View results
```
